## MCP Server Design

### Core Tools to Implement
1. **nova_solve_problem** - Main problem-solving interface
   - Parameters: problem, domains, max_iterations, model
   - Returns: Nova Process results

2. **nova_get_session_status** - Check session status
   - Parameters: session_id
   - Returns: Session status and progress

3. **nova_list_sessions** - List all active sessions
   - Parameters: None
   - Returns: List of session IDs and statuses

4. **nova_get_session_result** - Get session results
   - Parameters: session_id
   - Returns: Final results if available

5. **nova_cancel_session** - Cancel running session
   - Parameters: session_id
   - Returns: Cancellation status

### Architecture
- Standalone MCP server (not tied to Gradio)
- Uses NovaSystem core components
- Supports both stdio and HTTP transport
- Async/await compatible
- Error handling and logging
